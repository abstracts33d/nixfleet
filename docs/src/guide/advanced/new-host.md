# Adding a New Host

Step-by-step guide for adding a new machine to the config.

## 1. Add the Host to `fleet.nix`

Open `modules/fleet.nix` and add an `mkHost` entry to the `hosts` list:

```nix
(mkHost {
  hostName = "my-host";
  org = abstract<username>;          # organization (sets shared defaults)
  platform = "x86_64-linux";   # or "aarch64-linux", "aarch64-darwin"
  hardwareModules = [
    ./_hardware/my-host/hardware-configuration.nix
    ./_hardware/my-host/disk-config.nix
  ];
  hostSpecValues = {
    hostName = "my-host";
    useNiri = true;
    isDev = true;
    isImpermanent = true;
  };
})
```

For VMs, set `isVm = true` and omit `hardwareModules` (defaults are provided by `mkVmHost`):

```nix
(mkHost {
  hostName = "my-vm";
  org = abstract<username>;
  platform = "x86_64-linux";
  isVm = true;
  hostSpecValues = {
    hostName = "my-vm";
    isMinimal = true;
  };
})
```

`mkFleet` automatically calls the right constructor (`mkNixosHost`, `mkVmHost`, or `mkDarwinHost`) based on the `platform` and `isVm` flag.

## 2. Add Hardware Config

Create `modules/_hardware/my-host/` with:
- `hardware-configuration.nix` — generated by `nixos-generate-config`
- `disk-config.nix` — disko partitioning (use a template from `_shared/disk-templates/`)

VMs using default QEMU hardware do not need this directory.

## 3. Set Host Flags

Only declare what makes this host special. Smart defaults handle the rest:
- `useNiri = true` automatically enables `isGraphical` and `useGreetd`
- `isMinimal = true` disables `isGraphical` and `isDev`

Org defaults (username, GitHub credentials, timezone, locale, theme, SSH keys, GPG key, password file paths) are inherited from `mkOrg` and do not need repeating in `hostSpecValues`. Org-level modules (agenix, org-specific packages, Homebrew brews/casks) are also injected automatically via `mkOrg nixosModules`/`darwinModules`/`hmModules`.

## 4. Add Secrets

In the `secrets repo` repo:
1. Add the host's public key to `secrets.nix`
2. Re-encrypt secrets: `agenix -r`
3. Update: `nix flake update secrets`

## 5. Test

```sh
# Files must be tracked by git before evaluation
git add .

# Verify it evaluates
nix eval .#nixosConfigurations.my-host.config.system.build.toplevel.name

# Build it
nix build .#nixosConfigurations.my-host.config.system.build.toplevel

# Full validation
nix run .#validate
```

## 6. Install

```sh
nix run .#install -- --target root@<ip> -h my-host -u <username>
```
