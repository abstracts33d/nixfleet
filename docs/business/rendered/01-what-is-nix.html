Qu'est-ce que Nix ?

Une introduction non technique a la technologie qui fonde NixFleet — trois couches, un ecosysteme, des garanties mathematiques.

Document confidentiel · NixFleet · Europe · MMXXVI

Le mot « Nix » designe trois technologies distinctes qui fonctionnent ensemble. Comprendre cette distinction est essentiel pour saisir pourquoi NixFleet est possible — et pourquoi aucun outil concurrent ne peut reproduire ses garanties.

01 Couche 1 — Fondation

Nix, le langage

Nix est un langage de programmation fonctionnel pur. Concretement, cela signifie :

  • Pas de variables mutables — une valeur declaree ne change jamais. Il n'y a pas de « mise a jour » d'une variable.
  • Pas d'effets de bord — le resultat d'une expression Nix depend uniquement de ses entrees, jamais de l'etat du systeme ou du moment de l'execution.
  • Determinisme — la meme expression Nix, evaluee sur n'importe quelle machine, produit toujours le meme resultat.
Imaginez une recette de cuisine ou chaque ingredient est pese au milligramme pres, ou la temperature est controlee au degre pres, et ou le resultat est garanti identique a chaque fois que vous la suivez. C'est ce que le langage Nix fait pour l'infrastructure informatique.

Implication business

La description de votre infrastructure est mathematiquement reproductible. Quand un ingenieur ecrit une configuration Nix, le resultat n'est pas une approximation — c'est une garantie. Deux machines qui partagent la meme configuration sont identiques, bit pour bit, aujourd'hui ou dans trois ans.

02 Couche 2 — Logiciels

Nix, le gestionnaire de paquets

Au-dessus du langage, Nix est un gestionnaire de paquets universel — le plus grand au monde.

  • Isolation des paquets — chaque paquet vit dans un chemin unique, determine par le hash SHA-256 de toutes ses dependances. Deux versions coexistent sans conflit. Une mise a jour ne peut physiquement pas casser un autre logiciel.
  • Reproductibilite — un fichier flake.lock epingle la version exacte de chaque paquet et de chaque dependance transitive. Le meme ensemble de logiciels est installe, partout, toujours.
  • Rollback instantane — chaque operation cree une nouvelle « generation ». Revenir a l'etat precedent est instantane — il suffit d'activer la generation precedente. Pas de desinstallation, pas de risque.

Devshells — l'environnement reproductible

Au lieu de demander a chaque developpeur d'installer manuellement ses outils, un fichier Nix declare l'environnement complet :

# Environnement de developpement (conceptuel)
devshell :
  - Python     3.11.8
  - PostgreSQL 15.6
  - Redis      7.2.4
  - Outils     formatters, linters, tests

Un nouveau developpeur execute une seule commande et obtient exactement le meme environnement que tous ses collegues. Plus de « ca marche sur ma machine ». Plus de journees perdues a installer des dependances.

03 Couche 3 — Systeme

NixOS, le systeme d'exploitation

NixOS est un systeme d'exploitation Linux entierement configure en Nix. C'est la ou la puissance du langage et du gestionnaire de paquets atteint son plein potentiel.

Tout est declare

Un seul fichier de configuration declare l'integralite du systeme :

  • Noyau Linux et ses modules
  • Services systeme — SSH, pare-feu, serveur web, base de donnees
  • Utilisateurs et leurs permissions
  • Configuration reseau
  • Paquets installes
  • Partitionnement disque via disko

L'impermanence — la securite par l'oubli

Au redemarrage, tout l'etat non explicitement declare est efface. Le systeme de fichiers root est remis a zero. Un malware qui s'installe dans le systeme est automatiquement supprime. Toute modification non autorisee est effacee. L'etat reel correspond toujours a sa declaration.

Chaque couche herite des garanties de la couche inferieure. Le systeme d'exploitation est reproductible parce que le gestionnaire de paquets est reproductible. Le gestionnaire de paquets est reproductible parce que le langage est deterministe. C'est une chaine de garanties ininterrompue, du code source jusqu'au systeme deploye.
